📖 Agent Technical Guide & Shared Task Board API

Tài liệu hướng dẫn và đặc tả API dành cho các Agent tự động hóa. Kết nối các phiên làm việc độc lập của Agent với Cổng điều phối bằng cách đọc trạng thái, làm việc thực tế và gửi báo cáo về Dashboard.

1. Khái Niệm Dòng Việc Độc Lập (Independent Agent Workflow)

Vì mỗi Agent là một phiên chat Antigravity độc lập, dòng việc được đồng bộ hóa thông qua Dashboard chạy cục bộ:
  • PM Giao Việc: PM click vào nút điều phối. Dashboard gọi POST /api/task-state và đặt trạng thái task thành "Assigned".
  • Agent Kiểm Tra (Đọc tiến độ): Agent trong phiên chat của mình sử dụng các công cụ để đọc trạng thái thông qua GET /api/task-state.
  • Agent Làm Thật: Agent chạy các lệnh build cục bộ (như Godot CLI, Selenium, Git CLI) trực tiếp trong cửa sổ chat của mình.
  • Agent Báo Cáo Thật: Sau khi hoàn thành, Agent cập nhật trạng thái nhiệm vụ thành "Completed" qua API POST /api/task-state và gửi báo cáo ngắn gọn lên Web qua API POST /api/logs.

2. Chi Tiết API Cục Bộ (Local HTTP API Reference)

GET /api/task-state - Lấy trạng thái hiện tại của tất cả công việc. 
curl http://localhost:8080/api/task-state
POST /api/task-state - Cập nhật trạng thái một công việc. 
curl -X POST -H "Content-Type: application/json" -d '{"task":"coding-source", "status":"Completed"}' http://localhost:8080/api/task-state
POST /api/logs - Gửi dòng nhật ký nâng cấp trực tiếp lên bảng của Owner. 
curl -X POST -H "Content-Type: application/json" -d '{"task":"Bàn giao Source Code", "agent":"Coding Agent", "details":"Kiểm tra assets và chuẩn bị mã nguồn sạch, chuẩn hóa bus layout.", "status":"Hoàn thành"}' http://localhost:8080/api/logs
