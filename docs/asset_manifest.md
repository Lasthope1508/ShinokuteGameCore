# Quantum Starter Asset Manifest

Every reused, generated, edited, or imported asset in the reskin must be listed here before it is placed in production scenes.

Default skin size and runtime-size baseline lives in `res://docs/default_skin_size_ssot.md`. That file is the first SSOT for deciding whether a Candy Sky Islands asset is a full replacement, wrapper over legacy geometry, pending, unused, audio-replaced, or deferred.

Owner-supplied 2D player character conversion lives in `res://docs/reskin_2d_character_to_3d_runbook.md`. Required production flow: Photoroom full owner source, polygon source-pose extraction, 9Router reference-based turnaround/multiview sprite generation, Photoroom full generated sheet, polygon extraction for each sprite/view, then 3D reconstruction/render.

Canonical cross-platform asset rule: every Candy Sky Islands runtime platform uses the same canonical asset keys from this manifest and theme SSOT. iOS, Android, HTML5, and Roblox may have platform-specific file derivatives for size/format/runtime needs, but those derivatives must be recorded against the same key and must not become separate art/design branches.

Runtime payload hygiene rule: production exports use selected runtime resources only. Authoring, raw, legacy, candidate, docs, debug, tools, output, root `models`, root `meshes`, and `assets/themes/candy_sky_islands/source` folders must stay hidden from Godot import with `.gdignore` unless a listed production scene explicitly needs the asset. Legacy paths in this manifest or `theme_config.tres` are evidence/measurement references, not export dependencies. Runtime Candy GLB `.import` files keep `meshes/generate_lods=false` to avoid Godot LOD warnings from non-finite normals and to keep Web PCK size lower for this low-poly stylized game.
