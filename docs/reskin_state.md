## Canonical Core/Asset Standard

Candy Sky Islands now records the single Shinokute reskin standard:

- Reusable function behavior belongs in `ShinokuteGameCore`; Candy may wrap/configure/connect it but must not fork it into game scenes.
- UI skin/layout SSOT and asset checklist are mandatory before UI, HUD, function skin, or platform work.
- `docs/asset_manifest.md`, `docs/default_skin_size_ssot.md`, and `Resources/Data/Themes/candy_sky_islands/theme_config.tres` are the Candy canonical sources for asset keys, old-to-new role mapping, layout, text owner regions, fonts, audio routes, and in-game size.
- iOS, Android, HTML5, and Roblox must consume the same canonical Candy asset keys and art/audio/source assets.
- Platform-specific code belongs only in platform layers/branches/shims for input, safe area, export shell, renderer, storage bridge, or API glue.
- Platform-specific generated files are derivatives of canonical asset keys, not separate design branches.
- If a future platform needs a different size or format, record the transform and target platform against the same asset key instead of inventing a new design role.

- Settings core wiring and Candy function-skin UI applied on 2026-07-10: Shinokute core owns settings defaults/persistence/signals; Candy owns `CandySettingsPanel`, BGM/SFX routes, and game camera shift-lock behavior. Candy Shift Lock is the owner-requested lock-behind-character mode: camera locks behind the visual character face axis; character facing drives camera yaw, not mouse/camera yaw driving the character. Local movement behavior: `move_back` backpedals, while `move_left`/`move_right` turn the character and camera in the player-facing camera view instead of strafing sideways; for the current CHR077 `+Z` face axis, this means `move_right` subtracts yaw and `move_left` adds yaw.
